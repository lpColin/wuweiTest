# 五维测评系统 - 硬件规格说明书

> 版本：v1.0 | 日期：2026-06-29 | 规模：200 人同时在线

---

## 一、概述

本规格说明书针对 **五维测评系统** 在 **200 人同时在线** 场景下的硬件配置需求进行设计。系统基于 B/S 架构，包含 Web 前端、API 后端、数据库、缓存、对象存储等组件。

### 1.1 业务负载估算

| 指标 | 估算值 | 说明 |
|---|---|---|
| 同时在线用户（峰值） | 200 人 | 考虑早晚高峰 |
| 日活用户（DAU） | 800 - 1,200 人 | 在线率约 20% |
| 单次测评 API 请求量 | 60 - 300 次/人 | 每题一次提交 |
| 峰值 QPS（API） | 60 - 100 req/s | 答题集中提交场景 |
| 题库存储量 | 5,000+ 题 | 含文本、图片附件 |
| 用户数据量（年） | 约 2 GB | 用户表 + 测评记录 |
| 附件存储（年） | 约 20 GB | 海报、头像、文章封面等 |

### 1.2 系统组件

| 组件 | 技术选型 | 部署方式 |
|---|---|---|
| 前端 Web | React / Vue 3 + Vite | 静态资源部署至 Nginx |
| 后端 API | Node.js（NestJS / Express） | PM2 多进程 |
| 关系数据库 | MySQL 8.0+ / PostgreSQL 14+ | 主从架构 |
| 缓存 | Redis 6.x+ | 单机或 Sentinel |
| 对象存储 | 阿里云 OSS / MinIO | 独立服务 |
| 反向代理 | Nginx / OpenResty | SSL 终结 + 负载均衡 |

---

## 二、部署架构

由于 200 人同时在线规模较小，采用**单层应用 + 主从数据库 + 缓存**的轻量架构，无需微服务拆分。
## 三、服务器硬件规格

### 3.1 应用/Web 服务器（3 节点）

| 项目 | 规格 |
|---|---|
| **用途** | 运行 Web 前端静态资源 + API 后端（Node.js） |
| **数量** | 3 台 |
| **CPU** | 2 核（x86_64，2.0 GHz+） |
| **内存** | 4 GB |
| **系统盘** | 40 GB SSD |
| **数据盘** | 无需额外数据盘 |
| **网络** | 千兆网卡 |
| **操作系统** | CentOS 7.9+ / Ubuntu 20.04+ |
| **部署内容** | Nginx + Node.js（PM2 管理） |
| **单台月估费** | ~¥120 - ¥150（云服务器） |

**说明：** 3 节点通过 Nginx upstream 做负载均衡，单节点故障不影响服务。200 人同时在线下单节点承载约 70 峰值请求，CPU 和内存余量充足。

### 3.2 反向代理 / 负载均衡

| 项目 | 规格 |
|---|---|
| **用途** | 反向代理、SSL 终结、静态资源缓存、负载均衡 |
| **部署方式** | 复用 3.1 中一台节点，无需独立硬件 |
| **软件** | Nginx / OpenResty |

### 3.3 数据库服务器 - MySQL 主库

| 项目 | 规格 |
|---|---|
| **用途** | 用户数据、题库、测评记录、积分等核心业务数据 |
| **数量** | 1 台 |
| **CPU** | 4 核（x86_64，2.5 GHz+） |
| **内存** | 8 GB |
| **系统盘** | 40 GB SSD |

| **数据库版本** | MySQL 8.0+ / PostgreSQL 14+ |
| **关键参数** | innodb_buffer_pool_size = 5 GB，max_connections = 500 |
| **单台月估费** | ~¥400 - ¥600（云数据库 RDS 或自建） |

### 3.4 Redis 缓存服务器

| 项目 | 规格 |
|---|---|

| **数量** | 1 台 |
| **CPU** | 2 核 |
| **内存** | 4 GB（建议 maxmemory 2 GB） |
| **系统盘** | 40 GB SSD |
| **网络** | 千兆网卡 |

| **策略** | allkeys-lru 淘汰策略 |

**缓存内容估算：** 用户 Session（~20 MB）+ 抽题缓存（~100 MB）+ 其他（~30 MB），峰值小于 500 MB。

| **年增量** | ~20 GB |
| **月估费** | ~¥10（按量计费）|

## 五、硬件成本汇总

| 资源 | 规格 | 数量 | 月估费 |

## 六、性能指标与验收标准

| 指标 | 目标值 | 测试方式 |
|---|---|---|
| 页面首屏加载时间 | ≤ 2 秒 | Lighthouse / Chrome DevTools |
| API 响应时间（P95） | ≤ 500 ms | wrk / autocannon 压测 |
| 随机抽题响应 | ≤ 200 ms | 依赖 Redis 缓存命中率 ≥ 95% |
| 测评提交处理耗时 | ≤ 1 秒 | 含评分计算和存储写操作 |
| 报告生成耗时 | ≤ 3 秒 | 含雷达图渲染 |
| 并发支撑能力 | ≥ 200 同时在线的稳定运行 | JMeter / Locust 压测 |
| 系统可用性 | ≥ 99.9%（年度） | 多节点容错 + 健康检查 |
| 数据备份 RPO | ≤ 24 小时 | 每日全量备份 + Binlog |
| 数据恢复 RTO | ≤ 2 小时 | 从库切换或备份恢复 |

## 七、扩展性说明

当在线用户数增长时，可按以下顺序扩展：

| 在线用户 | 操作 | 月估费用 |
|---|---|---|
| 200 | 基础配置（如上方案 B） | ~¥1,400/月 |
| 500 | Web 节点增至 5 台，MySQL 加 1 从库 | ~¥2,500/月 |
| 1,000 | Web 节点 8 台（4C8G），Redis 集群，MySQL 读写分离 | ~¥5,000/月 |
| 5,000 | 微服务拆分，分库分表，Redis Cluster | ~¥15,000+/月 |
